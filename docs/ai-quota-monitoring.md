# AI用量监控系统

## 概述
实现了AI用量监控系统，能够每小时自动抓取 `https://ai-quota.fintopia.tech/users/27` 页面的用量数据，并在cc-pet-web应用的设置面板中提供可视化展示。

## 功能特性

### 后端功能
1. **定时抓取服务** - 每小时自动访问目标URL并抓取AI用量数据
2. **数据库存储** - 将抓取的数据存储在SQLite数据库的`ai_quota_history`表中
3. **REST API** - 提供三个API端点：
   - `GET /api/quota/current` - 获取最新用量数据
   - `GET /api/quota/history` - 获取历史用量数据
   - `GET /api/quota/stats` - 获取用量统计信息

### 前端功能
1. **标签页界面** - 在设置面板中添加了"AI用量"标签页
2. **实时数据显示** - 展示当前用量百分比、已用量和总量
3. **进度条可视化** - 用颜色编码的进度条显示用量情况（绿色<60%，黄色60-80%，红色>80%）
4. **历史图表** - 提供图表显示用量历史趋势，支持24小时、7天、30天时间范围
5. **统计数据** - 显示最高、平均、最低用量等统计信息

## 配置

### 环境变量
系统通过环境变量配置AI用量监控功能：
- `AI_QUOTA_COOKIE` - 用于访问目标站点的认证cookie（remember_token的值）

启动服务器时需要设置此环境变量：
```bash
AI_QUOTA_COOKIE="57|95378471c0d9cd82fd89f5da39e27f11cc318c867797313661469ed5bd72645fae8e2e41755c64af694f9782c9ae8a96837b3d3e737c3382bcefec5ec2998022" pnpm dev:server
```

## 数据库结构

新增了`ai_quota_history`表用于存储历史用量数据：
- `id` - 主键，自增长
- `timestamp` - 数据记录的时间戳
- `usage_data` - JSON格式的用量数据
- `raw_content` - 原始页面内容（可选，用于调试）

## 技术实现

### 后端技术
- 使用`axios`进行HTTP请求
- 使用`cheerio`解析HTML页面
- SQLite数据库存储历史数据
- Fastify框架提供API服务

### 前端技术
- React组件实现UI界面
- SVG图表展示历史趋势
- 自定义时间范围选择功能

## 使用方法

1. 启动服务器时设置`AI_QUOTA_COOKIE`环境变量
2. 启动cc-pet-web应用
3. 在设置面板中点击"AI用量"标签页
4. 查看当前用量和历史趋势图表

系统将自动每小时抓取数据，前端每分钟刷新显示最新的用量信息。

## 注意事项

- 首次启动后需要等待第一个小时才能看到用量数据
- 目标网站可能需要认证，确保障号cookie有效
- 图表会自动显示最近的数据，支持不同的时间范围查看
